Extension Development Guide

Build live features for MindSpring — by hand or by asking the AI.

1. Overview

MindSpring extensions are small, self-contained packages that add features to the app at runtime. Each extension lives in its own directory inside extensions/ and consists of a manifest plus optional frontend files (JavaScript, CSS).

Extensions can be created two ways:

  • Ask the AI — tell the MindSpring AI to "build me a word count panel" and it generates, deploys, and hot-loads the extension in one step.
  • Write by hand — create the files in extensions/your-id/ and restart the app (or call the build API).
Zero build step. The frontend is vanilla JavaScript — no bundler, no transpiler. Extension scripts load directly in the browser via <script> tags.

2. Extension Structure

extensions/
  word-count/
    manifest.json       # required — metadata and entry points
    panel.js            # optional — frontend JavaScript
    styles.css          # optional — scoped CSS

All files must be in the extension's root directory (no subdirectories). Filenames cannot contain .. or /.

3. Manifest Format

{
  "id":          "word-count",
  "name":        "Word Count",
  "description": "Shows word and character counts for the active note",
  "version":     "1.0.0",
  "enabled":     true,
  "frontend": {
    "panel":   "panel.js",
    "styles":  "styles.css"
  }
}
FieldTypeDescription
idstringUnique identifier. Lowercase alphanumeric + dashes only (/^[a-z0-9-]+$/).
namestringHuman-readable display name.
descriptionstringShort description shown in Settings.
versionstringSemver version string.
enabledbooleanSet to false to disable without deleting.
frontend.panelstringFilename of the JS file to load (loaded as a regular script, not a module).
frontend.stylesstringFilename of the CSS file to load.

4. The inkwell API

Every extension has access to the window.inkwell global object. This is the only interface between extensions and the core app.

MethodReturnsDescription
inkwell.registerPanel({ id, title, render }) void Register a panel that appears in the Extensions side panel. render(container) receives a DOM element to populate.
inkwell.registerFooterWidget({ id, render }) void Register a widget for the editor footer area.
inkwell.on(event, callback) unsubscribe fn Listen to a lifecycle event (see table below). Returns a function you can call to stop listening.
inkwell.getActiveNote() object | null Returns { id, title, body, created_at, updated_at } for the currently selected note, or null.
inkwell.getNotes() array Returns a copy of all decrypted notes in memory.
inkwell.refreshPanel(id) void Re-render a specific panel by calling its render() function again.
inkwell.notify(message, type) void Show a toast notification. type is 'info' (default) or 'error'.
inkwell.api(path, opts) Promise Make an authenticated API call. Same signature as fetch but includes credentials and JSON headers. Path is relative to /api.

5. Lifecycle Events

EventPayloadFires when
noteSelect{ id, title, body, ... }A note is selected in the sidebar.
noteSave{ id, title, body, ... }A note is saved (after the 700ms debounce).
noteCreate{ id, title, body, ... }A new note is created.
noteDeleteid (string)A note is deleted. Payload is just the note ID.
notesLoad[{ id, title, body, ... }]All notes have been decrypted and loaded after login.

Subscribe to events to react to user actions and keep your panel content up to date.

6. Registering Panels

Panels appear as collapsible sections inside the Extensions side panel (the panel icon in the sidebar). Each panel gets a container element to render into.

inkwell.registerPanel({
  id: 'word-count',
  title: 'Word Count',
  render(container) {
    const note = inkwell.getActiveNote();
    if (!note) {
      container.innerHTML = '<em>Select a note</em>';
      return;
    }
    const words = note.body.split(/\s+/).filter(Boolean).length;
    const chars = note.body.length;
    container.innerHTML = `Words: ${words} &middot; Characters: ${chars}`;
  }
});

// Re-render when the user switches notes or saves
inkwell.on('noteSelect', () => inkwell.refreshPanel('word-count'));
inkwell.on('noteSave',   () => inkwell.refreshPanel('word-count'));
Important: panel.js is loaded as a regular <script>, not an ES module. Do not use import/export. Access everything through the inkwell global.

7. Full Examples

Example 1: Reading Time Estimator

manifest.json

{
  "id": "reading-time",
  "name": "Reading Time",
  "description": "Estimates how long the current note takes to read",
  "version": "1.0.0",
  "enabled": true,
  "frontend": { "panel": "panel.js" }
}

panel.js

inkwell.registerPanel({
  id: 'reading-time',
  title: 'Reading Time',
  render(el) {
    const note = inkwell.getActiveNote();
    if (!note) { el.innerHTML = '<em>No note selected</em>'; return; }
    const words = note.body.split(/\s+/).filter(Boolean).length;
    const mins = Math.max(1, Math.round(words / 200));
    el.innerHTML = `About ${mins} min read (${words} words)`;
  }
});

inkwell.on('noteSelect', () => inkwell.refreshPanel('reading-time'));
inkwell.on('noteSave',   () => inkwell.refreshPanel('reading-time'));

Example 2: Note Stats Dashboard

panel.js — shows total note count and most recently updated note.

inkwell.registerPanel({
  id: 'note-stats',
  title: 'Note Stats',
  render(el) {
    const notes = inkwell.getNotes();
    if (!notes.length) {
      el.innerHTML = '<em>No notes yet</em>';
      return;
    }
    const totalWords = notes.reduce((sum, n) =>
      sum + (n.body || '').split(/\s+/).filter(Boolean).length, 0);
    const latest = notes[0];
    el.innerHTML = `
      <div><strong>${notes.length}</strong> notes</div>
      <div><strong>${totalWords.toLocaleString()}</strong> total words</div>
      <div>Latest: ${latest.title || 'Untitled'}</div>
    `;
  }
});

inkwell.on('notesLoad',  () => inkwell.refreshPanel('note-stats'));
inkwell.on('noteCreate', () => inkwell.refreshPanel('note-stats'));
inkwell.on('noteDelete', () => inkwell.refreshPanel('note-stats'));
inkwell.on('noteSave',   () => inkwell.refreshPanel('note-stats'));

Example 3: Extension with Custom CSS

panel.js

inkwell.registerPanel({
  id: 'tag-cloud',
  title: 'Tag Cloud',
  render(el) {
    const notes = inkwell.getNotes();
    const freq = {};
    notes.forEach(n => {
      (n.body || '').match(/#\w+/g)?.forEach(tag => {
        freq[tag] = (freq[tag] || 0) + 1;
      });
    });
    const tags = Object.entries(freq).sort((a, b) => b[1] - a[1]);
    el.innerHTML = tags.map(([tag, count]) =>
      `<span class="tc-tag">${tag} <small>${count}</small></span>`
    ).join(' ') || '<em>No #tags found in notes</em>';
  }
});

inkwell.on('notesLoad', () => inkwell.refreshPanel('tag-cloud'));
inkwell.on('noteSave',  () => inkwell.refreshPanel('tag-cloud'));

styles.css

.tc-tag {
  display: inline-block;
  background: #e9e3d6;
  padding: 0.2em 0.5em;
  border-radius: 3px;
  font-family: "JetBrains Mono", monospace;
  font-size: 0.78rem;
  margin: 0.15em;
}
.tc-tag small { opacity: 0.5; }
CSS tip: Prefix your class names with a short identifier (like tc- above) to avoid conflicts with MindSpring's core styles or other extensions.

8. AI Builder Mode

The easiest way to create extensions is to ask the MindSpring AI directly. The AI has full knowledge of the extension API and will generate, deploy, and hot-load extensions in one step.

Just open the AI panel and describe what you want:

  • "Build me a word count panel that updates live"
  • "Create an extension that shows a tag cloud from #hashtags in my notes"
  • "Build a panel that lists all notes sorted by length"
  • "Make a pomodoro timer widget"

The AI generates the extension code, the backend writes it to the extensions/ directory, and the frontend hot-loads it — all automatically. You'll see a toast notification when the extension is installed, and it will appear in the Extensions panel immediately.

To modify an existing AI-built extension, just ask: "Update the word count panel to also show paragraph count."

9. Management REST API

All routes are under /api/extensions. Management routes (POST, PATCH, DELETE) require authentication.

MethodPathAuthDescription
GET/api/extensionsNoList enabled extensions (frontend loader uses this).
GET/api/extensions/:id/files/:filenameNoServe an extension's frontend file (JS, CSS).
GET/api/extensions/admin/listYesList all extensions including disabled ones.
POST/api/extensions/buildYesDeploy an extension. Body: { id, name, description, files: { "panel.js": "...", ... } }
PATCH/api/extensions/:idYesUpdate manifest. Body: { enabled: false }
DELETE/api/extensions/:idYesPermanently delete an extension and all its files.

Deploy via curl

curl -X POST http://localhost:8080/api/extensions/build \
  -H "Content-Type: application/json" \
  -H "Cookie: token=YOUR_JWT" \
  -d '{
    "id": "hello-world",
    "name": "Hello World",
    "description": "A minimal test extension",
    "files": {
      "panel.js": "inkwell.registerPanel({ id: \"hello\", title: \"Hello\", render(el) { el.textContent = \"Hello from an extension!\"; } });"
    }
  }'

10. Tips & Constraints

  • No ES modules. Extension scripts are loaded via <script src="...">, not <script type="module">. Don't use import/export.
  • No build step. Write vanilla JavaScript. No JSX, no TypeScript, no bundler required.
  • Scope your CSS. Use a unique prefix for all class names to avoid collisions.
  • Keep it small. Extensions should be focused and self-contained. One panel, one purpose.
  • Error isolation. If your render() function throws, the error is caught and displayed in the panel — it won't crash the app.
  • Persistence. Extensions are stored in a Docker volume and survive container restarts. They persist across docker compose down unless you use -v to remove volumes.
  • Security model. Extensions run with full access to decrypted note data in the browser. Only install extensions you trust. Self-hosted users control everything.
  • No backend extensions (yet). The current system supports frontend-only extensions. Backend route extensions are reserved for a future version.
